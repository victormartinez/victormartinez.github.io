Dados geolocalizados com Redis

Jan 20, 2022 5 min leitura

TL;DR; Vamos usar o Redis para armazenar dados geolocalizados.

Quando falamos em Redis invariavelmente pensamos em cache. Porm ele evoluiu e hoje considerado um servidor que armazena diferentes estruturas de dados (hashes, sets, lists, etc).

No post de hoje vamos sair do uso comum e entender de que forma o Redis nos ajuda com quando precisamos trabalhar dados geolocalizados (para mais info, vide documentao).

Configurao do ambiente

Para replicar os comandos voc vai precisar configurar um ambiente local. Estou usando o docker e o redis-cli. O CLI vem com a instalao do Redis mas eu gosto de t-lo separado (veja este tutorial). Ento, basta executar os comandos abaixo:

$ docker run --name=my-redis --rm -p "6379:6379" redis:6 redis-server
$ redis-cli

127.0.0.1:6379>

Estrutura de Dados

Armazenar qualquer coisa no Redis requer a utilizao de ndices. De forma simples, uma chave string que aponta para alguma estrutura de dados. Para lidar com dados geolocalizados o Redis utiliza a estrutura de sorted set (conjunto ordenado) de forma que cada ponto (lat/long) rotulado com uma string. A imagem abaixo ilustra de forma simplificada essa arquitetura:

No exemplo, criamos um ndice chamado Salvador (cidade em que sou nascido e criado ) que aponta para um conjunto de pontos.

Ateno: como o Redis utiliza a estrutura de conjuntos no possvel existir no mesmo ndice dois pontos com o mesmo nome. A insero de um ponto com o mesmo nome corresponde a uma atualizao das coordenadas (operao de upsert).

Mos obra

O Redis nos fornece uma API com os seguintes comandos: GEOADD,GEODIST, GEOHASH, GEOPOS, GEOSEARCH. Vamos iniciar adicionando alguns pontos ao ndice location:salvador:

Ateno: Os comandos logo sero depreciados: GEOMEMBER, GEORADIUS, GEORADIUS_RO, GEORADIUSBYMEMBER, GEORADIUSBYMEMBER_RO. Por isso, no vamos explor-los neste post.

127.0.0.1:6379> GEOADD location:salvador -38.5139209 -12.9730385 mercadomodelo -38.5329599 -13.0101531 faroldabarra -38.5260635 -13.0107505 morrodocristo

Aqui, duas observaes importantes:

  1. O redis recebe primeiro a longitude e depois a latitude.

    1. CMD <set-name> <long> <lat> <member-name>
  2. Como vamos manipular os pontos dentro do ndice eu optei por adotar um padro lowercase nas strings e no ndice. Isso evita erros na busca e tambm torna suficientemente legvel os dados armazenados (veja a seo Redis keys da documentao).

Uma vez inseridos os pontos, podemos brincar com a API. Vamos a alguns casos de uso.

Obter longitude e latitude de um ou mais pontos

127.0.0.1:6379> GEOPOS location:salvador faroldabarra

1) 1) "-38.53296071290969849"
   2) "-13.01015213126540715"
2) 1) "-38.52607816457748413"
   2) "-13.0107807421129209"

Calcular a distncia, em metros, entre dois pontos

127.0.0.1:6379> GEODIST location:salvador faroldabarra morrodocristo

"749.1386"

OBS: Voc pode passar, no final, a mtrica de distncia que voc quer (m, km, ft, mi).

Atualizar um ponto

127.0.0.1:6379> GEOADD location:salvador -38 -12 elevadorlacerda

127.0.0.1:6379> GEOADD location:salvador -38.5159605 -12.9740495 elevadorlacerda

127.0.0.1:6379> GEOPOS location:salvador elevadorlacerda

OBS: Como estamos trabalhando com conjuntos a atualizao de um ponto se d por inserir um ponto com a mesma string (itens repetidos no so permitidos).

Buscar

# Pontos em um raio de 1600 metros a uma determinada longitude/latitude
127.0.0.1:6379> GEOSEARCH location:salvador FROMLONLAT -38.5175291 -13.0060716 BYRADIUS 1600 m
1) "morrodocristo"

# Pontos em um raio de 200 km de um ponto existente
127.0.0.1:6379> GEOSEARCH location:salvador FROMMEMBER faroldabarra BYRADIUS 200 km
1) "faroldabarra"
2) "morrodocristo"
3) "elevadorlacerda"
4) "mercadomodelo"

# A mesma query acima porm com as coordenadas e a distncia
127.0.0.1:6379> GEOSEARCH location:salvador FROMMEMBER faroldabarra BYRADIUS 200 km WITHCOORD WITHDIST
1) 1) "faroldabarra"
   2) "0.0000"
   3) 1) "-38.53296071290969849"
      2) "-13.01015213126540715"
2) 1) "morrodocristo"
   2) "0.7491"
   3) 1) "-38.52607816457748413"
      2) "-13.0107807421129209"
3) 1) "elevadorlacerda"
   2) "4.4180"
   3) 1) "-38.51596087217330933"
      2) "-12.97405009779294005"
4) 1) "mercadomodelo"
   2) "4.6149"
   3) 1) "-38.51392239332199097"
      2) "-12.97303874405036339"

Observaes:

  • GEOSEARCH tambm permite restringir a quantidade de resultados por meio do parmetro COUNT.
  • Substitua o BYRADIUS 200 km por BYBOX 200 200 km e o Redis ir considerar um retngulo ao invs de uma rea circular.
  • possvel retornar os valores ordenados pela distncia. Basta adicionar ASC ou DESC ao final da busca.

Armazenar os resultados da busca

O redis permite que voc faa uma busca e armazene em um ndice especfico. No exemplo abaixo buscamos em location:salvador e armazenamos em location:results:salvador.

127.0.0.1:6379> GEOSEARCHSTORE location:results:salvador location:salvador FROMMEMBER faroldabarra BYRADIUS 200 km

127.0.0.1:6379> KEYS *
1) "location:salvador"
2) "location:results:salvador"

127.0.0.1:6379> ZRANGE location:results:salvador 0 -1
1) "faroldabarra"
2) "morrodocristo"
3) "mercadomodelo"

Geohash

Redis usa a representao em Geohash, string de 11 caracteres, para representar os pontos no conjunto de forma que ela pode ser traduzida em uma coordenada (lat / long). 

127.0.0.1:6379> GEOHASH location:salvador morrodocristo faroldabarra
1) "7jstgr6nvs0"
2) "7jstgpt4kg0"

Voc pode verificar esse mapeamento geohash lat/long por meio do site https://geohash.org/.

Operaes com Sorted Set

Como o Redis armazena essa estrutura na forma de sorted sets ento possvel acessar esses dados usando APIs dessa estrutura de dados. Vejamos alguns exemplos (todos os comandos esto na documentao):

127.0.0.1:6379> ZRANGE location:salvador 0 -1
1) "faroldabarra"
2) "morrodocristo"
3) "elevadorlacerda"
4) "mercadomodelo"

127.0.0.1:6379> ZREM location:salvador elevadorlacerda
(integer) 1

127.0.0.1:6379> ZCARD location:salvador
(integer) 3

127.0.0.1:6379> ZCOUNT location:salvador -inf +inf
(integer) 3

Consideraes Finais

  • Pense no Redis alm da chave-valor. Existem outras estruturas de dados para voc brincar.
  • Cuidado com o nome do ndice e dos pontos. Voc vai precisar busc-los de uma forma inteligente e que evite erros de digitao.
  • Antes de inserir todos os pontos pense em como voc vai buscar os dados. Ser que faz sentido todos os pontos estarem no mesmo ndice? E se voc precisar fazer sharding?
